# Framework Limitations & Known Blindspots

**框架局限性与已知盲区**

This document is an honest assessment of what the Contrarian Opportunity Analysis System can and cannot do. It is written by the framework's designer, not generated by AI, and should be read before interpreting any output from the system.

本文档是对逆向机会分析系统能力和局限性的诚实评估。在解读系统的任何输出之前，应先阅读本文档。

---

## 1. Backtest Hindsight Bias / 回测后见之明偏差

**The most critical limitation.**

All historical backtest judgments (C1-C6) were made by a human who already knew what happened after each test date. When we labeled the COVID crash bottom (2020-03-23) as C2=accumulating, we already knew the Fed would launch unlimited QE. When we labeled the 2022 bear market bottom (2022-10-12) as C2=accumulating, we already knew inflation would peak and markets would recover.

This means the backtest validates the framework's **structural logic** (signal classification, layer synthesis, mislocation detection) but does NOT validate the framework's **predictive power**. The 100% accuracy rate on 3-month BUY signals is a property of the structural logic working correctly with perfect inputs — not evidence that the system will achieve 100% accuracy with real-time inputs.

**所有历史回测判断都是在已知未来信息的情况下做出的。** 100%的准确率验证的是框架的结构逻辑，不是预测能力。只有 `daily_scan_history.json` 中带时间戳的实时前瞻性判断才能验证真正的预测能力。

**What would change this:** The daily auto-scanner running from March 2026 onward produces genuine forward-looking judgments. Only after 3+ months of daily data with verified outcomes will we have statistically meaningful evidence of real predictive power.

---

## 2. Signal Definition Ambiguity / 信号定义模糊

The six criteria use qualitative judgments: "Is this domain aligned with the macro trend?" — but "aligned" is not a precisely defined term. Two analysts looking at the same market data could reasonably disagree on whether C1 should be 1 or 0.

The framework reduces complex, multi-dimensional market conditions to binary states (0 or 1). This compression is by design — it forces a clear judgment — but it also means edge cases are poorly handled. A market that is "mostly aligned but with significant headwinds" gets the same C1=1 as a market that is "perfectly aligned with no headwinds."

**信号定义依赖定性判断，不同分析者可能对同一市场数据给出不同的二进制判断。** 框架将复杂情况压缩为0/1，边界情况处理不好。

**What would change this:** Establishing quantitative thresholds for each criterion (e.g., "C1=1 if 200-day moving average slope > 0 AND sector relative performance > -10%") would reduce ambiguity but also reduce the framework's ability to incorporate qualitative, structural insights that pure quantitative models miss.

---

## 3. Insufficient Sample Size / 样本量不足

The SPY extended backtest covers 25 events over 18 years. The metals backtest covers 21 events. For any single asset, extreme market events (crashes, panics, euphoria peaks) occur only a few times per decade.

With single-digit samples per event type, a 100% hit rate has no statistical significance. A coin flip could achieve similar results over 5-8 trials. The confidence interval on the true accuracy rate is extremely wide.

**每个品种遇到的极端行情只有几次，小样本下的100%胜率没有统计意义。** 只有持续积累大量实时判断数据才能建立统计显著性。

**What would change this:** The daily auto-scanner generates ~20 judgments per week across 4 tickers. After 3 months, we'll have ~260 data points. After 6 months, ~520. This is the minimum needed to start making statistically meaningful claims.

---

## 4. No Signal Timing Mechanism / 没有信号时机机制

The framework can answer "Should I buy X right now?" but it cannot answer "When should I ask that question?" 

In backtests, we pre-selected historically significant moments (crash bottoms, euphoria peaks, crisis events). In real-time use, the daily auto-scanner runs every day regardless of market conditions, which partially solves this — but it means most daily scans will be in "normal" market conditions where the framework's contrarian logic has less edge.

The framework is designed for moments of dislocation, but it has no built-in mechanism to detect when a dislocation is occurring.

**框架能判断"现在该不该买"，但不能判断"什么时候该问这个问题"。** 回测中的时刻点是事后选择的。每日自动扫描部分解决了这个问题，但大部分交易日市场处于"正常"状态，框架的逆向逻辑优势不明显。

**What would change this:** Adding a volatility/dislocation detector (e.g., VIX > 25, or price deviation > 2 standard deviations from mean) to trigger enhanced analysis on abnormal days, while reducing scan frequency on normal days.

---

## 5. No Position Sizing or Risk Management / 没有仓位管理

The framework outputs categorical signals (BUY / WATCH / HOLD / AVOID) but says nothing about:

- How much to buy (position sizing)
- When to add to a position (scaling in)
- When to reduce a position (scaling out)
- Where to set stop-losses (risk limits)
- How to manage a portfolio of multiple signals (correlation risk)

A correct BUY signal that leads to a 20% gain is useless if the investor allocated only 1% of their portfolio. A correct BUY signal is destructive if the investor went all-in and the position dropped 15% before recovering.

**框架只输出分类信号，不涉及仓位大小、分批建仓、止损设置、组合管理。** 正确的信号配合错误的仓位管理仍然可能导致亏损。

**What would change this:** Integrating a position sizing module that considers signal strength (binary code patterns), portfolio concentration, and user-defined risk tolerance. This is a planned future feature but does not exist in v0.1.

---

## 6. LLM Input Quality Dependency / 依赖LLM输入信息质量

When running in automated mode (daily_scan.py), all six criteria judgments are made by Claude based on web search results. This introduces several failure modes:

- **Search bias:** Web search results may over-represent recent sentiment and under-represent structural factors.
- **Narrative capture:** Claude may be influenced by dominant media narratives rather than contrarian analysis.
- **Data gaps:** If critical information is behind paywalls, not indexed, or in languages Claude doesn't search well, judgments will be based on incomplete data.
- **Model consistency:** Different Claude API calls may produce different judgments for the same market conditions on the same day.

The framework itself has no mechanism to verify the quality or completeness of the information Claude uses to make judgments.

**所有判断依赖Claude搜索到的信息质量。** 搜索偏差、媒体叙事影响、数据缺失、模型输出不一致性都可能导致判断错误，框架本身无法校验输入信息的质量。

**What would change this:** Adding multiple data sources (financial APIs for quantitative data, multiple LLM providers for cross-validation, sentiment indices for independent verification) would reduce single-source dependency. Cross-referencing Claude's qualitative judgments with quantitative indicators (RSI, moving averages, put/call ratios) would provide a sanity check layer.

---

## Summary / 总结

This framework is a structured thinking tool, not an oracle. It helps organize complex market analysis into a consistent, auditable format. Its value is in the process — forcing the analyst to consider six distinct dimensions across three layers — not in the output signal itself.

The daily auto-scanner and auto-verifier are designed to build a genuine, forward-looking track record over time. Until that track record reaches statistical significance (minimum 6 months of daily data with verified outcomes), all claims about the framework's predictive power should be treated with appropriate skepticism.

**这个框架是一个结构化思维工具，不是预言机。** 它的价值在于过程——迫使分析者从六个维度、三个层次来考虑问题——而不在于输出的信号本身。在实时验证数据达到统计显著性之前，对框架预测能力的所有声明都应保持适当的怀疑态度。
